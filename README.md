# Frozen Assets - Band Website

Official website for **Frozen Assets**, a classic rock covers band based in Tokyo, Japan.

**Live:** [frozen-assets.band](https://frozen-assets.band)

## Tech Stack

- **Frontend:** React 18, React Router 6, date-fns
- **Build:** Vite 5
- **Hosting:** Vercel
- **Data Source:** BandChat API (synced to static JSON)
- **Fonts:** Oswald (headings), Inter (body) via Google Fonts

## Project Structure

```
frozen/
  index.html              # Entry point
  vite.config.js          # Vite config (port 5174, API proxy)
  vercel.json             # Vercel SPA rewrites
  scripts/
    sync.js               # BandChat data sync script
  src/
    main.jsx              # React entry, global CSS imports
    App.jsx               # Routes and lazy loading
    services/
      api.js              # Public (static JSON) and Admin (BandChat) API clients
    hooks/
      usePublicApi.js     # Data fetching hook with loading/error states
      useScrollReveal.js  # IntersectionObserver scroll animations
    context/
      AuthContext.jsx     # Admin authentication state
    components/
      layout/
        Layout.jsx        # Main layout (Header + Outlet + Footer + Trivia)
        Header.jsx        # Navigation with scroll/mobile states
        Footer.jsx        # Footer + FooterBar (version, legal links)
      pages/              # All public pages (Home, Shows, About, Songs, etc.)
      features/           # Reusable components (GigCard, MemberCard)
      trivia/             # Trivia popup system
      admin/              # Admin panel (dashboard, content, guests, blog, merch, AI agents)
  styles/
    variables.css         # Design tokens (colors, fonts, spacing)
    global.css            # Reset, base styles, utility classes
    layout.css            # Header, footer, navigation
    hero.css              # Hero section
    animations.css        # Keyframes and animation classes
    responsive.css        # Media queries
    pages/                # Per-page styles (archive, gig-detail, setlists, etc.)
    components/           # Per-component styles (gig-card, member-card, trivia-popup)
    admin/                # Admin panel styles
  public/
    data/                 # Static JSON data (generated by sync.js)
    images/               # Static images, icons, favicon
```

## Getting Started

### Prerequisites

- Node.js 18+
- npm

### Install

```bash
npm install
```

### Configure

Copy `.env.example` to `.env` and fill in values:

```bash
cp .env.example .env
```

Required variables:
- `VITE_BANDCHAT_URL` - BandChat API URL
- `VITE_WORKSPACE_ID` - Frozen Assets workspace ID
- `SYNC_*` variables - Credentials for the sync script
- `RESEND_API_KEY` - Resend API key for contact form emails
- `RESEND_FROM_EMAIL` - (optional) From address once domain is verified (defaults to `onboarding@resend.dev`)

### Sync Data

Pull latest data from BandChat:

```bash
node scripts/sync.js
```

This writes static JSON files to `public/data/` that the website reads at runtime.

### Development

```bash
npm run dev
```

Runs at [http://localhost:5174](http://localhost:5174).

### Build

```bash
npm run build
```

Output goes to `dist/`.

### Preview Production Build

```bash
npm run preview
```

## Data Flow

```
BandChat API  -->  sync.js  -->  public/data/*.json  -->  React app (fetch at runtime)
```

The website is a fully static SPA. All dynamic data is pre-synced to JSON files. No runtime backend needed for public pages. The admin panel authenticates directly against BandChat for management tasks.

## Versioning

```bash
npm run version:patch   # 1.0.3 -> 1.0.4
npm run version:minor   # 1.0.3 -> 1.1.0
npm run version:major   # 1.0.3 -> 2.0.0
```

The version is injected at build time via `__APP_VERSION__` and displayed in the footer bar.

## Deployment

Deployed on Vercel. Push to `master` triggers automatic deployment. The `vercel.json` config handles SPA routing (all non-API routes serve `index.html`).
